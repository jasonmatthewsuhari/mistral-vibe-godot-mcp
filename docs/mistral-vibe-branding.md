# Mistral Vibe Branding Notes

Use this message consistently across docs/pitches:

> "A Godot MCP server built for Mistral Vibe: terminal-native agent tooling for real game-dev automation."

## Branding Pillars
1. **Vibe-native workflow**
   - MCP tools exposed for terminal/agent usage.
2. **Agent delegation ready**
   - Supports multi-agent/subagent task splits for rapid iteration.
3. **Practical outcome**
   - Real Godot actions: run/debug, scene editing, UID ops, render capture/interact.
4. **Safe local execution**
   - Localhost-only bridge and token-based auth.
